# Contributing to Solo Founder OS

> **Short version**: this is a paid product, not a community fork. But ideas / bug reports / suggestions are very welcome via Issues.

---

## What contributions are welcome

### ✅ Issues (anyone)
- Bug reports — agent / skill / command / hook not working as documented
- Documentation typos / unclear sections in INSTALL / README / CLAUDE.md
- Compatibility issues — specific Claude Code version that breaks something
- Use-case feedback — "this skill would be 10x more useful if it also did X"

### ✅ Discussions (paid buyers only)
- New agent / skill / command requests
- Workflow patterns you discovered using the pack
- Q&A about specific configurations
- Show & tell — what you built using this pack

### ⚠️ Pull Requests (rare · case-by-case)
- Bug fixes for hooks (JS files) — yes, MIT-licensed code
- Doc clarifications — usually yes
- New agents / skills / commands — usually no (we maintain editorial control to keep the pack coherent · feel free to fork for your own use under CC-BY-NC)

---

## How to file a good issue

```markdown
**What you did**: ran `/plan` on a Next.js project
**What you expected**: planner outputs phased plan
**What actually happened**: planner exits after 3 lines with error "..."
**Environment**: Claude Code v0.X · macOS Sequoia · ~/.claude/ default install
```

The more concrete, the faster I can fix.

---

## How NOT to contribute

- ❌ Don't open a PR that copies the pack to a different repo and adds your branding (license violation)
- ❌ Don't open a PR rewriting half the agents in your preferred style — fork your own version
- ❌ Don't request "translate to language X" — bilingual is on the roadmap (v2.0)

---

## Communication

- **GitHub Issues**: bug reports / suggestions (public)
- **Email** (paid buyers): founder@example.com
- **Founder tier 1:1 call** (Founder pack purchasers): book within 30 days of purchase

---

## Code of conduct

Be excellent to each other. The default tone here is direct technical feedback,
not corporate niceties. But personal attacks · spam · self-promotion in issues
will get you banned without warning.

---

## License clarity (so PRs are legally clean)

By submitting a PR, you agree your contribution is licensed under:
- **MIT** if it's code (hooks, scripts)
- **CC BY-NC 4.0** if it's docs / agents / skills / commands

You retain copyright on your contributions, but grant the project a perpetual
right to redistribute under the project's licenses.
