# Examples · What your `docs/` will look like in 1-3 months

Most Claude Code config packs give you **abstract patterns**. This folder gives you **filled-in artifacts** — what a real founder's `docs/` directory actually looks like after running the pack for a few months.

Names are sanitized (replaced with placeholders like *Acme Corp* / *ProjectAlpha*). Numbers, structure, and tone are real.

## How to read these

These are **outputs**, not templates. Don't copy them — they'll be wrong for your situation. Read them to:
- See what "filled" looks like vs the empty templates
- Steal the *level of detail* (not the content)
- Understand which fields you actually use vs ignore

After 1 month of running the pack, your own files will look more like these and less like the placeholder templates.
